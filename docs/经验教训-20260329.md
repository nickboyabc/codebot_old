# 经验教训总结 (2026-03-29)

## 一、认证系统问题

### 1. CORS 配置冲突
**文件**: `backend/config.py`

**问题**:
```python
allow_origins: ["*"]      # 通配符
allow_credentials: True    # 与 * 冲突！
```
当 `Access-Control-Allow-Credentials: true` 时，`Access-Control-Allow-Origin` 不能是通配符 `*`，否则浏览器会拒绝发送认证信息。

**解决**:
```python
allow_credentials: bool = False
```

**为什么难发现**: 本地开发环境（127.0.0.1）可能不受 CORS 预检请求影响，只有在某些浏览器或部署环境下才暴露。

---

### 2. JWT Secret 持久化问题
**文件**: `backend/core/auth.py`

**问题**: `_get_jwt_secret()` 会生成随机密钥并保存到 `data/.jwt_secret`。如果该文件丢失或路径变化，所有已有 token 失效。

**解决**: 确保 `data` 目录持久化，或通过环境变量设置固定的 `CODEBOT_JWT_SECRET`。

---

### 3. Token 类型验证问题
**文件**: `backend/api/deps.py`

**问题**: `verify_token()` 验证 token 类型 (`access` vs `refresh`)，如果前端用 refresh_token 调用需要 access_token 的接口会失败。

**解决**: 前端确保使用正确的 token 类型。

---

## 二、前端认证状态问题

### 4. 导航栏 el-menu 缺少 `:router="true"`
**文件**: `frontend/src/components/NavMenu.vue`

**问题**: `el-menu` 没有设置 `:router="true"`，点击菜单项不会触发 Vue Router 导航。

**解决**:
```vue
<el-menu mode="horizontal" :default-active="$route.path" :router="true">
```

**为什么难发现**: 官方原版 codebot 没有这个组件，是自定义开发的。

---

### 5. App.vue 启动时未同步用户信息
**文件**: `frontend/src/App.vue`

**问题**: `onMounted` 中只获取通知数，不调用 `fetchCurrentUser()`。如果 localStorage 中 userInfo 缺失或过期，admin 菜单永远不会显示。

**解决**:
```javascript
onMounted(() => {
  userStore.fetchCurrentUser()  // 添加
  // ...
})
```

---

### 6. 401 响应拦截器的循环调用
**文件**: `frontend/src/utils/request.js`

**问题**: 401 时清理 token 并跳转登录页，但 `isLoggingOut` 标志只有 100ms，可能导致并发 401 时循环调用。

**解决**: 增加 `isLoggingOut` 标志的有效时间，或使用 Promise-based 互斥锁。

---

## 三、技能调度问题

### 7. OpenCode 技能目录路径错误
**文件**: `backend/core/tool_dispatcher.py`

**问题**: 只扫描 `~/.agents/skills/`，但用户技能保存在 `~/.config/opencode/skills/`。

**解决**: 同时扫描两个目录：
```python
oc_dir = Path.home() / ".agents" / "skills"
oc_config_dir = Path.home() / ".config" / "opencode" / "skills"
```

---

### 8. 技能匹配阈值问题
**文件**: `backend/core/tool_dispatcher.py`

**问题**: 内置技能阈值 0.20 太低，且中文使用 unigram 分词导致匹配不精准。

**解决**: 考虑使用 bigram 或调整阈值。

---

### 9. 定时任务误判
**文件**: `backend/api/routes/chat.py`

**问题**: `_looks_like_schedule_message()` 的逻辑有缺陷，`task_verbs` 检查被提前返回跳过。

**解决**: 禁用定时任务功能（函数直接返回 False）。

---

## 四、开发建议

### 10. 官方原版 vs 修改版
官方原版 `codebot-main` 没有认证系统、用户管理、NavMenu 组件等。这些是我们添加的自定义功能。

**建议**: 维护清晰的变更日志，记录每个自定义功能的添加原因和依赖。

---

### 11. 认证功能添加时的注意点
添加认证系统到无认证项目时，需要：
- [ ] 修改 `router/index.js` 添加路由守卫
- [ ] 修改所有受保护 API 添加 `Depends(get_current_user)`
- [ ] 修改前端 request 拦截器添加 Authorization header
- [ ] 修改 App.vue 添加登录状态判断
- [ ] 创建 NavMenu.vue 组件
- [ ] 处理 CORS 配置

---
